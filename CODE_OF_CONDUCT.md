# Code of Conduct

This project follows a simple rule: be respectful, practical, and safety-minded.

## Expected Behavior

- Be kind to beginners.
- Explain tradeoffs clearly.
- Keep discussions focused on making the tool safer and easier to understand.
- Redact secrets, private paths, hostnames, and local network details.
- Assume mistakes are fixable.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Sharing another person's secrets or private machine details.
- Pressuring maintainers or users to run unsafe commands.
- Posting exploit details publicly instead of using the security reporting path.

## Enforcement

Maintainers may edit, hide, or remove comments and close issues or pull requests that make the project unsafe or hostile.
